# Pretraining data

> The trillion-token web + code + math + books corpora that pretrain a frontier model. What's in them, how they're built, and what's hard.

## The canonical sources

| Source | Order of magnitude (tokens) | Notes |
| --- | --- | --- |
| Common Crawl | ~250T raw, ~10T after dedup + quality | The bulk of the web |
| Wikipedia | ~5B per language | High signal, clean |
| GitHub / The Stack | ~3T deduplicated code | Filtered for permissive licenses |
| arXiv | ~50B | Math + science |
| Books (legal questions) | ~30B | Long-form prose |
| StackExchange | ~10B | High-quality Q&A |
| PubMed | ~5B | Biomedical |
| Project Gutenberg | ~3B | Out-of-copyright literature |
| Multilingual web (CC-{lang}) | ~50B per major language | Filtered per-language CC slices |
| Synthetic | growing fast | Generated by frontier models |

Frontier corpora (RedPajama [Computer, 2023](https://together.ai/blog/redpajama)[^redpajama]; Dolma [Soldaini et al., 2024](https://doi.org/10.48550/arXiv.2402.00159)[^dolma]; FineWeb [Penedo et al., 2024](https://doi.org/10.48550/arXiv.2406.17557)[^fineweb]) publish their recipes openly; closed-lab corpora (Llama-3, DeepSeek-V3, Claude, GPT-4) are closely held.

## The pipeline at a glance

```
raw HTML / PDF / EPUB / WARC
       │
       ▼  trafilatura / resiliparse / per-format parsers
extracted text
       │
       ▼  fastText / CLD3
language-tagged
       │
       ▼  quality classifier + heuristics
quality-filtered
       │
       ▼  MinHash + LSH near-dedup
deduplicated
       │
       ▼  PII / harmful-content scrub
safety-filtered
       │
       ▼  domain mixing weights
shard manifest
       │
       ▼  tokenize + pack
final training shards
```

Each step throws out 50–99% of the bytes that go in. The art is in *which* bytes.

## Quality filtering

The two dominant approaches:

1. **Heuristic** — line lengths, repetition rates, ratio of letters/digits/punctuation, presence of boilerplate ("javascript required"). Easy to implement; brittle at the edges. C4 [Raffel et al., 2020](https://doi.org/10.48550/arXiv.1910.10683)[^c4] is the canonical heuristic-filtered corpus.
2. **Classifier-based** — train a tiny classifier on "good" vs "bad" text. "Good" = a high-quality reference (Wikipedia, books). FineWeb-Edu uses a Llama-3-judged classifier; DataComp-LM [Li et al., 2024](https://doi.org/10.48550/arXiv.2406.11794)[^dclm] scaled this approach and showed it dominates pure heuristics.

A classifier-filtered corpus of ~3T tokens can match a heuristic-filtered corpus of ~15T tokens on downstream eval — the most impactful "scaling law" of recent years is on data quality, not data quantity.

## Deduplication

[Lee et al., 2022](https://doi.org/10.18653/v1/2022.acl-long.577)[^dedup] showed that deduplication is among the most impactful single steps in data preparation. Models trained on deduped data:

- Memorise less.
- Reach a given loss with fewer steps.
- Generalise better.

Methods:

- **Exact**: hash documents (or paragraphs). O(N).
- **Near-dedup**: MinHash signatures + locality-sensitive hashing (LSH). Catches paraphrases and templated content.
- **Semantic**: embed and cluster. Expensive; usually only on a final QA pass.

The 250T raw Common Crawl shrinks to ~10T after near-dedup. That ratio is consistent across multiple groups' reports.

## Domain mixing

Modern corpora upweight high-value domains:

| Domain | Typical weight (mid-2026) |
| --- | --- |
| Web (filtered) | 60–80% |
| Code | 5–25% (frontier models are coding-heavy) |
| Math / arXiv / textbooks | 2–10% |
| Books | 2–10% |
| Multilingual | varies (5–30% non-English) |

These weights are tuned by ablation: small-model proxy runs vs eval benchmarks. The Llama-3 [paper](https://doi.org/10.48550/arXiv.2407.21783) describes the methodology in detail.

## Curriculum and annealing

A trend since Phi-1 / Phi-2: end pretraining with a "cooldown" or "annealing" phase on a small, very-high-quality subset. This is responsible for several percentage points on MMLU at no compute cost. See [MiniCPM](https://doi.org/10.48550/arXiv.2404.06395)[^minicpm] for a transparent recipe.

## Contamination

A model is "contaminated" if benchmark test items leaked into pretraining. Contamination inflates eval scores misleadingly. Mitigations:

- **N-gram contamination check** — for each benchmark item, search the corpus for matching n-grams (typically 13-grams).
- **Decontamination** — remove matched documents from the corpus.
- **Held-out benchmarks** — benchmarks released after the model's pretraining cutoff (LiveBench [White et al., 2024](https://doi.org/10.48550/arXiv.2406.19314)[^livebench]).

Frontier model reports now include contamination analyses for major benchmarks. Always check before believing a "+5 on MMLU" claim.

## Licensing, ethics, and copyright

This is a fast-moving legal landscape; the engineering-relevant short version:

- **Books3** and other shadow-library corpora have been removed from public datasets post-litigation.
- **Code** under permissive licenses (MIT, Apache, BSD) is widely included; copyleft (GPL) is increasingly excluded or attributed.
- **Opt-out signals** (`robots.txt`, `ai.txt`) are honoured by major crawlers — but historical CC dumps predate this.
- **Personal data** — EU GDPR and US state laws are increasingly enforced; pretraining data should be scrubbed for emails, phone numbers, addresses, and unique identifiers.

If you're building pretraining or continued-pretraining data, get legal review *before* the run, not after.

## What "small" pretraining looks like

Most teams will never pretrain a 70B model. Many will do "continued pretraining" on a small domain corpus — adding 1B–50B tokens of medical, legal, or company-specific text on top of an open base model. The same pipeline applies, scaled down by 1000×.

For that workflow: pick a base model with a license that allows continued pretraining (Llama 3, Mistral, Qwen, Gemma), use the same tokenizer, follow the same filtering/dedup/mix recipe, and budget a learning rate at ~10–30% of the original peak. See [Fine-tuning](../../fine-tuning/index.md).
